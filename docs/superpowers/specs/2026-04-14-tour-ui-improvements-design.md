# Tour UI Improvements — Design Spec

4 независимых улучшения визуального поведения туров.

---

## 1. Затемнение фона (overlay_enabled)

### Цель
Дать возможность на уровне тура отключать затемнение фона при воспроизведении шагов.

### Изменения

**Миграция:**
- Колонка `overlay_enabled` (boolean, default: `true`, not null) в таблице `onboard_on_rails_tours`

**Админка (форма тура):**
- Чекбокс "Затемнение фона" в секции визуальных настроек, по умолчанию отмечен
- Добавить `overlay_enabled` в `tour_params`

**API:**
- Поле подхватывается автоматически — `Api::ToursController` отдаёт все атрибуты тура

**Клиент (`tour_renderer.js`):**
- В `createOverlay()`: если `tour.overlay_enabled === false` — создавать прозрачный оверлей (`background: transparent`) вместо полупрозрачного
- Прозрачный оверлей сохраняет возможность закрытия тура по клику вне тултипа

**Локализация:**
- Добавить перевод лейбла чекбокса в `en.yml` и `ru.yml`

---

## 2. Минимальная ширина шага (min_width)

### Цель
Дать возможность на уровне шага задавать минимальную ширину тултипа/модалки. Работает для всех тем.

### Изменения

**Модель Step:**
- Без миграции — новый ключ `min_width` (integer, пиксели) в существующем jsonb `style_overrides`

**Админка (форма шага):**
- Числовой инпут "Мин. ширина (px)" в секции Style overrides
- Placeholder с текущим дефолтным значением
- Добавить `min_width` в разрешённые ключи `style_overrides` в `step_params`

**Клиент (`theme_engine.js`):**
- При наличии `min_width` в overrides — устанавливать CSS-переменную `--oor-step-min-width`

**CSS (`client.css`):**
- `.oor-tour-step`: добавить `min-width: var(--oor-step-min-width, auto)`

**Локализация:**
- Добавить перевод лейбла в `en.yml` и `ru.yml`

---

## 3. Скрытие навигации для одношаговых туров

### Цель
Не показывать навигационные точки (или счётчик), если в туре только 1 шаг.

### Изменения

**Клиент (`tour_renderer.js`, `createTooltip`):**
- Обернуть блок рендеринга навигации (точки / счётчик) в условие `totalSteps > 1`
- Если `totalSteps === 1` — навигация не рендерится

Без изменений в моделях, API или админке.

---

## 4. Настройка шрифта по умолчанию (default_font)

### Цель
Позволить разработчику задать дефолтный шрифт для всех туров через конфигурацию движка. Шрифт (например, Rubik) подключается хост-приложением, движок только применяет.

### Изменения

**Конфигурация (`lib/onboard_on_rails/configuration.rb`):**
- Новый атрибут `default_font`, по умолчанию `nil`
- При `nil` используется текущий системный стек: `-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif`
- Пример: `config.default_font = "Rubik"`

**Передача в клиент (`meta_tags_helper.rb`):**
- Мета-тег `onboard-on-rails-default-font` со значением `OnboardOnRails.configuration.default_font`
- Если `nil` — мета-тег не рендерится

**Клиент (`theme_engine.js`):**
- При инициализации читать мета-тег `onboard-on-rails-default-font`
- Если задан — устанавливать `--oor-step-font` как дефолтное значение
- Step-level `font_family` из `style_overrides` по-прежнему перебивает дефолт

**Форма шага (dropdown шрифтов):**
- Добавить "Rubik" в список доступных шрифтов

**Локализация:**
- Добавить перевод "Rubik" в dropdown (если нужен)

## Порядок реализации

Все 4 задачи независимы и могут выполняться параллельно. Рекомендуемый порядок при последовательной работе:

1. Скрытие навигации (1 строчка)
2. Затемнение фона (миграция + чекбокс + клиент)
3. Минимальная ширина (style_overrides + инпут + CSS)
4. Шрифт по умолчанию (конфиг + мета-тег + клиент)
